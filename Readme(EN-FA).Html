🔬 What is Psiphon Tracker "DEY Warrior"?

🔬 Psiphon Tracker "DEY Warrior" چیست؟

Psiphon Tracker V 1.0 "DEY Warrior" is a multi-stage, thread-parallel proxy scanner designed for high-latency, heavily censored environments (like Iran). Every single test is performed through the proxy itself – the scanner never uses its own internet connection to judge a proxy's health. This means it works perfectly even when DNS, ICMP, and all foreign websites are blocked.

It accurately emulates Psiphon's real connection protocols (OSSH seed exchange with anti-echo, Meek POST with obfuscated headers, Fronted TLS to real CDNs) based on reverse engineering. Each proxy receives a fair, multi-axis score (Gen, PSI, Generic Viability) and is classified from DEAD to FULLY_READY.

A two-phase network watchdog (hardcoded IPs + sampling your own proxy list) keeps the scan alive even when every external IP goes dark. Manual pause/resume with P/R keys and --max-alive auto-stop give you full control.

Psiphon Tracker V 1.0 "DEY Warrior" یک اسکنر پروکسی چندمرحله‌ای و چندنخی است که برای محیط‌های با تأخیر بالا و سانسور شدید (مانند ایران) طراحی شده است. تک‌تک تست‌ها از مسیر خود پروکسی انجام می‌شوند – اسکنر هرگز از اتصال اینترنت خود برای قضاوت درباره سلامت پروکسی استفاده نمی‌کند. به همین دلیل حتی وقتی DNS، ICMP و تمام وب‌سایت‌های خارجی مسدود باشند، کاملاً کار می‌کند.

این اسکنر پروتکل‌های واقعی Psiphon (تبادل بذر OSSH با ضد-اکو، ارسال Meek با هدرهای مبهم‌سازیشده، Fronted TLS به CDNهای واقعی) را بر اساس مهندسی معکوس شبیه‌سازی می‌کند. هر پروکسی یک امتیاز منصفانه چندمحوره (Gen، PSI، Generic Viability) دریافت می‌کند و از DEAD تا FULLY_READY طبقه‌بندی می‌شود.

نگهبان شبکهٔ دومرحله‌ای (IPهای ثابت + نمونه‌گیری از لیست پروکسی خودتان) اسکن را حتی وقتی همهٔ IPهای خارجی قطع شوند زنده نگه می‌دارد. توقف/ادامهٔ دستی با کلیدهای P/R و توقف خودکار با --max-alive کنترل کامل را به شما می‌دهند.

🚀 Quick Start with Launcher (Windows)

🚀 شروع سریع با Launcher (ویندوز)

A professional Windows batch launcher (launcher.bat) is included. It manages all settings and runs scans in the same window so P/R keys work.

How to use:

  1. Double-click launcher.bat
  2. Press [1] to set your proxy list file (e.g. my_ips.txt)
  3. Choose a scan mode:
    • Q – Quick Scan (20 threads, 30s timeout, --no-geo --no-diversity)
    • D – Deep Scan (50 threads, 90s timeout, --verbose)
    • F – Full Scan (50 threads, 120s timeout, --verbose)
    • C – Custom Scan (uses all configured settings)
  4. Press Y to start
  5. During scan: press P to pause, R to resume

The launcher saves your settings in launcher_settings.ini for next time.

🛡️ Two‑Phase Network Watchdog

🛡️ نگهبان شبکهٔ دومرحله‌ای

The scanner uses a unique two-phase watchdog to detect internet loss:

  1. Phase 1 – Well-known IPs: Checks TCP port 443 on several hardcoded IPs (Google, Cloudflare, GitHub, …). No DNS needed.
  2. Phase 2 – Proxy targets: If Phase 1 fails, it randomly samples proxies from your own list. If even one proxy responds, the internet is considered alive.

Only when both phases fail will the scanner pause and wait for the network to return. It automatically resumes when connectivity is restored, or you can press R to force a retry.

اسکنر از یک نگهبان دومرحله‌ای منحصربه‌فرد برای تشخیص قطعی اینترنت استفاده می‌کند:

  1. فاز ۱ – IPهای معروف: پورت ۴۴۳ چندین IP ثابت (گوگل، کلاودفلر، گیت‌هاب، varzesh3.com و …) را بررسی می‌کند. بدون نیاز به DNS.
  2. فاز ۲ – خود پروکسی‌ها: اگر فاز ۱ شکست خورد، از پروکسی‌های لیست خودتان نمونه‌برداری می‌کند. اگر حتی یک پروکسی پاسخ دهد، شبکه زنده است.

تنها زمانی که هر دو فاز شکست بخورند، اسکنر متوقف می‌شود و منتظر بازگشت شبکه می‌ماند. با بازیابی اتصال، به‌طور خودکار ادامه می‌یابد یا می‌توانید R را برای تلاش مجدد فشار دهید.

📈 Fair Scoring System

📈 سیستم امتیازدهی منصفانه

1. General Score (Gen, max 50)

Measures everyday proxy performance:

  • TLS tunnel success to b-cdn.net: +25
  • Slow TLS success (after retry): +15
  • Each reachable public endpoint (Google, Cloudflare, GitHub, Telegram Web, YouTube): +6
  • Successful 1 KB data transfer: +8
  • TCP hints (TTL < 64): +2
  • DNS over SOCKS: +2
  • Real browsing success (>50%): up to +10
  • Persistent connection (5 sequential requests): +2

2. PSI Score (Psiphon compatibility, max 55)

Measures Psiphon-specific compatibility:

  • Psiphon server list reachability: up to 30
  • TLS tunnel success: +5
  • OSSH seed exchange: +10 (PASS) / +5 (INCONCLUSIVE)
  • SSH banner detection: +4 (PASS) / +2 (INCONCLUSIVE)
  • Server list fetch: +10
  • Meek handshake: +6
  • UDP relay: +2
  • Port diversity (through proxy): up to +8
  • Advanced HTTP (POST/PUT/DELETE, Cookie, Referrer, Compression): +4
  • Censorship bypass success: up to +9 (3 sites)
  • Stability ≥ 80%: +3

3. Generic Viability (G_Via, max 50)

Usefulness for non-Psiphon tools like V2Ray/VPNs:

  • TCP open: +5
  • Forward proxy detected (HTTP_FWD): +5
  • Each reachable general endpoint: +4
  • Data transfer: +8
  • Advanced HTTP: +5

⚡ Advanced Internals

⚡ جزئیات فنی پیشرفته

  • Socket tuning: TCP_NODELAY on every socket.
  • TLS enforcement: Only TLS 1.2+ with AEAD ciphers, mandatory certificate verification.
  • Retry logic: Exponential backoff + jitter, plus per-test timeout multipliers (1.2×, 1.5×).
  • Atomic checkpoint: Write-temp-then-rename prevents corruption.
  • Thread safety: Locks protect shared caches and file writes; server list fetch uses atomic cache update.
  • Memory control: DNS and GeoIP caches expire after 300s; server list limited to 100 entries.
  • Zero DNS dependency: All tests use hostnames through the proxy; only SOCKS4 needs a pre-resolved IP (from cache or fallback).
  • Manual Pause/Resume: Keyboard listener thread with threading.Event, cross-platform.
  • Crash-resistant: All sockets & sessions properly closed in finally blocks.
  • تنظیم سوکت: TCP_NODELAY روی هر سوکت فعال است.
  • اجبار TLS: فقط TLS 1.2+ با رمزهای AEAD، تأیید اجباری گواهی.
  • منطق تلاش مجدد: پس‌روی نمایی + jitter، با ضریب‌های ۱.۲× و ۱.۵× زمان.
  • Checkpoint اتمیک: نوشتن در فایل موقت سپس بازنام‌گذاری – هرگز progress خراب نمی‌شود.
  • ایمنی ترد: قفل‌ها از کش‌های مشترک و نوشتن فایل‌ها محافظت می‌کنند؛ به‌روزرسانی کش لیست سرور اتمیک است.
  • کنترل حافظه: کش‌های DNS و GeoIP پس از ۳۰۰ ثانیه منقضی می‌شوند؛ لیست سرور به ۱۰۰ ورودی محدود است.
  • وابستگی صفر به DNS: تمام تست‌ها با نام دامنه از طریق پروکسی انجام می‌شوند؛ فقط SOCKS4 به IP از قبل حل‌شده (از کش یا فال‌بک) نیاز دارد.
  • توقف/ادامهٔ دستی: نخ شنوندهٔ کیبورد با threading.Event، چندسکویی.
  • مقاوم در برابر crash: تمام سوکت‌ها و نشست‌ها در بلوک‌های finally به‌درستی بسته می‌شوند.

🧩 Final Notes

🧩 نکات پایانی

  • Proxies that require authentication but none is provided are marked AUTH:Y and skipped (score 0).
  • Slow proxies get an automatic second chance and are labelled SLOW_OK.
  • Ping > 5000 ms triggers a HEAVY_LOAD warning but does not discard the proxy.
  • Every single test is performed through the proxy. No direct connections are used for scoring.
  • Output alive_proxies.txt contains 20+ tab-separated columns for deep analysis.
  • Press P to pause, R to resume during scan.
  • Python 3.7+ compatible. Tested on Linux, Windows, macOS.
  • پروکسی‌هایی که نیاز به احراز هویت دارند اما اعتبارنامه‌ای ارائه نشده، AUTH:Y علامت‌گذاری و رد می‌شوند (امتیاز ۰).
  • پروکسی‌های کند به‌طور خودکار شانس دوم می‌گیرند و برچسب SLOW_OK می‌خورند.
  • پینگ بالای ۵۰۰۰ میلی‌ثانیه هشدار HEAVY_LOAD می‌دهد اما پروکسی را حذف نمی‌کند.
  • تک‌تک تست‌ها از مسیر خود پروکسی انجام می‌شوند. هیچ اتصال مستقیمی برای امتیازدهی استفاده نمی‌شود.
  • خروجی alive_proxies.txt بیش از ۲۰ ستون جداشده با tab برای تحلیل عمیق دارد.
  • در حین اسکن P برای توقف، R برای ادامه.
  • سازگار با Python 3.7+. تست‌شده روی لینوکس، ویندوز، مک.
