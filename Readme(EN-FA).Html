🔬 Core Mission & Design

🔬 مأموریت اصلی و طراحی

PSI-Tracker V1.0 “Aegis Perfecta” is the ultimate proxy auditor for high‑latency, heavily censored environments. It tests every proxy through the proxy itself – the scanner never relies on its own internet to judge a proxy’s health. This means it works even when DNS, ICMP, and all foreign websites are blocked.

It accurately emulates Psiphon’s OSSH, Meek, and Fronted protocols, grades proxies with a fair multi‑axis score (Gen, PSI, Generic Viability), and includes real‑world tests like browsing, censorship bypass, stability, and persistent connection. A two‑phase network watchdog keeps the scan alive even when every external IP goes dark.

PSI-Tracker V1.0 “Aegis Perfecta” اسکنر نهایی پروکسی برای محیط‌های با فیلترینگ شدید و بدون DNS است. همهٔ تست‌ها از مسیر خود پروکسی انجام می‌شوند – اسکنر هرگز برای تأیید سلامت پروکسی به اینترنت خود متکی نیست. بنابراین حتی وقتی همهٔ سایت‌های خارجی مسدود باشند، اسکنر کار می‌کند.

این ابزار پروتکل‌های OSSH، Meek و Fronted سایفون را دقیقاً شبیه‌سازی می‌کند، پروکسی‌ها را با امتیاز چندمحوره ارزیابی می‌کند، و شامل تست‌های واقعی مثل مرور وب، عبور از فیلترینگ، پایداری و اتصال مداوم است. نگهبان شبکهٔ دومرحله‌ای اسکن را حتی وقتی همهٔ آی‌پی‌های خارجی قطع شوند زنده نگه می‌دارد.

🛡️ Two‑Phase Network Watchdog

🛡️ نگهبان شبکهٔ دومرحله‌ای

The scanner uses a unique two‑phase watchdog to detect internet loss:

  1. Phase 1 – Well‑known IPs: Checks TCP port 443 on several hardcoded IPs (Google, Cloudflare, GitHub, …). No DNS needed.
  2. Phase 2 – Proxy targets: If Phase 1 fails, it randomly samples proxies from your own list. If even one proxy responds, the internet is considered alive.

Only when both phases fail will the scanner pause and wait for the network to return.

اسکنر از یک نگهبان دومرحله‌ای برای تشخیص قطعی اینترنت استفاده می‌کند:

  1. مرحلهٔ ۱ – آی‌پی‌های معروف: پورت ۴۴۳ چندین آی‌پی سخت‌کد (گوگل، کلاودفلر، گیت‌هاب، …) را بررسی می‌کند. بدون نیاز به DNS.
  2. مرحلهٔ ۲ – خود پروکسی‌ها: اگر مرحلهٔ ۱ شکست خورد، از پروکسی‌های لیست شما نمونه‌برداری می‌کند. اگر حتی یک پروکسی پاسخ دهد، اینترنت زنده است.

تنها زمانی که هر دو مرحله شکست بخورند، اسکنر مکث می‌کند و منتظر بازگشت شبکه می‌ماند.

🚀 Usage Examples

🚀 مثال‌های استفاده

1. Basic scan

python psitracker.py --list proxies.txt --alive alive.txt --urls urls.txt

2. CIDR range with multiple ports

python psitracker.py --range 192.168.1.0/24 --port 1080 8080 --scope 192.168.0.0/16

3. Authentication

python psitracker.py --list auth_proxies.txt --auth admin:123456 --timeout 45

4. Resume previous scan

Progress saved in scan_progress.json. Use --retest to ignore it, --clean to reset.

5. Faster scanning (no GeoIP, no diversity)

python psitracker.py --list proxies.txt --no-diversity --no-geo

6. Stop after finding 20 alive proxies

python psitracker.py --list proxies.txt --max-alive 20

7. Pause/Resume manually

Press P to pause and R to resume anytime while scanning.

۳. احراز هویت

۴. ادامهٔ اسکن قبلی

پیشرفت در scan_progress.json ذخیره می‌شود. با --retest نادیده گرفته و با --clean پاک می‌شود.

۵. اسکن سریع‌تر (بدون GeoIP و تنوع پورت)

python psitracker.py --list proxies.txt --no-diversity --no-geo

📈 Fair Scoring & Classification

📈 امتیازدهی منصفانه و طبقه‌بندی

General Score (max 50) – measures everyday proxy usage:

  • TLS tunnel success: +25
  • Each reachable public endpoint (Google, Cloudflare, …): +6
  • Data transfer (1 KB): +8
  • TCP hints (TTL < 64): +2
  • DNS over SOCKS: +2
  • Real browsing: up to +10
  • Persistent connection (5 requests): +2

PSI Score (max 55) – measures Psiphon compatibility:

  • Psiphon server list reachability: up to 30
  • TLS tunnel success: +5
  • OSSH seed exchange: +10 (pass) / +5 (inconclusive)
  • SSH banner: +4 / +2
  • Server list fetch: +10
  • Meek handshake: +6
  • UDP relay: +2
  • Port diversity: up to +8
  • Advanced HTTP (POST/PUT/DELETE/…): +4
  • Censorship bypass: up to +9
  • Stability (≥80%): +3

Generic Viability (G_Via, max 50) – usefulness for non‑Psiphon tools like V2Ray.

⚡ Advanced Internals

⚡ جزئیات فنی پیشرفته

  • Socket tuning: TCP_NODELAY on every socket.
  • TLS enforcement: Only TLS 1.2+ with AEAD ciphers, mandatory certificate verification.
  • Retry logic: Exponential backoff + jitter, plus per‑test 1.2‑1.5× timeout multipliers.
  • Atomic checkpoint: Write‑temp‑then‑rename prevents corruption.
  • Thread safety: Locks protect shared caches and file writes; server list fetch uses atomic cache update.
  • Memory control: DNS and GeoIP caches expire after 300 s; server list limited to 100 entries.
  • Zero DNS dependency: All tests use hostnames through the proxy; only SOCKS4 needs a pre‑resolved IP (from cache or fallback).
  • Manual Pause/Resume: Keyboard listener thread with threading.Event, cross‑platform.
  • تنظیم سوکت: TCP_NODELAY روی هر سوکت فعال است.
  • اجبار TLS: فقط TLS 1.2+ با رمزهای AEAD، تأیید اجباری گواهی.
  • منطق تلاش مجدد: پس‌روی نمایی با لرزش، و ضرب‌های ۱.۲–۱.۵ برابری زمان.
  • نقطه بررسی اتمیک: نوشتن در فایل موقت سپس بازنام‌گذاری.
  • امنیت ترد: قفل‌ها از حافظه‌های مشترک و فایل‌ها محافظت می‌کنند؛ به‌روزرسانی کش لیست سرور اتمیک است.
  • کنترل حافظه: کش‌های DNS و GeoIP پس از ۳۰۰ ثانیه منقضی می‌شوند؛ لیست سرور به ۱۰۰ رکورد محدود است.
  • بدون وابستگی به DNS: تمام تست‌ها با نام دامنه از طریق پروکسی انجام می‌شوند؛ فقط SOCKS4 به IP از قبل حل‌شده نیاز دارد (از کش یا فال‌بک).
  • توقف/ادامهٔ دستی: نخ شنوندهٔ کیبورد با threading.Event، چندسکویی.

🧩 Final Notes

🧩 نکات پایانی

  • Proxies without credentials when needed are marked AUTH:Y and skipped (score 0).
  • Slow proxies get an automatic second chance and are labelled SLOW_OK.
  • Ping > 5000 ms triggers a HEAVY_LOAD warning but does not discard the proxy.
  • All tests are performed through the proxy; no direct connections are used for scoring.
  • Output alive_proxies.txt contains 20+ tab‑separated columns for deep analysis.
  • The scanner is Python 3.7+ compatible and has been tested on Linux / Windows / macOS.
  • Press P to pause, R to resume during scan.
  • پروکسی‌های بدون اعتبارنامه با AUTH:Y علامت‌گذاری و نادیده گرفته می‌شوند.
  • پروکسی‌های کند به‌طور خودکار شانس دوم می‌گیرند و برچسب SLOW_OK می‌خورند.
  • پینگ بالای ۵۰۰۰ میلی‌ثانیه هشدار HEAVY_LOAD می‌دهد اما پروکسی را حذف نمی‌کند.
  • تمام تست‌ها از مسیر خود پروکسی انجام می‌شوند؛ هیچ اتصال مستقیمی برای امتیازدهی استفاده نمی‌شود.
  • خروجی alive_proxies.txt بیش از ۲۰ ستون جداشده با تب برای تحلیل عمیق دارد.
  • اسکنر با Python 3.7 به بالا سازگار است و روی لینوکس/ویندوز/مک آزمایش شده است.
  • در حین اسکن با فشردن P توقف و با R ادامه دهید.
